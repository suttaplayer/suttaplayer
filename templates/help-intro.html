HELP

The help documentation has been organised into questions & answers for the purpose of brevity. The following answers should be sufficient to address 80% of the core features.

1. How to play/read a sutta?

A sutta can be played or read by:

  1. Direct URL browser request
    1. Enter the URL in the browser's address bar and press enter
  2. Goto URL in system clipboard
    1. Ensure the URL is copied into the system clipboard
    2. Click on the Share button (3 connected circles icon)
    3. Click on the Goto URL in system clipboard menu item

  3. Making a track selection from any one of the following originating lists within the App:

    1. Catalog list
    2. Search results list

    3. Playlist list

    4. Click the mouse (or tap) a blue sutta link reference in a list row. This will:

      1. Mount the origin list into the player for previous/next navigation within a playlist. An origin list badge denoting 'C' for Catalog, 'S' for Search and 'P' for Playlist will be displayed beside the Play/Pause button in the top control bar
      2. Load the nominated audio into the player for immediate play if auto play is toggled "on"
      3. Display the sutta text in the Reader view

2. Where are the Reader, Catalog, Search & Playlist page view tabs?

The suttaplayer app has a responsive user interface design which adjusts itself for different platforms, orientations and screen resolutions. On smaller (typically mobile) devices, the side menu can be expanded/collapsed via the side-menu toggle button.

  1. Click on the side-menu toggle button (3 horizontal bars icon) in the top control bar to reveal the navigation bar in the app

3. How to stop/start the audio from automatically playing when a track is loaded?

  1. Click the Audio button (speaker icon) in the top control bar to display the audio drop-down panel
  2. Toggle the Auto play button in the 4th row of the audio drop-down panel to switch on/off the audio Auto play feature

Note: Audio will be auto played when the Auto play button is toggled "on" and under any of the following scenarios:

  1. Via Browser's URL address bar
    1. When the app [running in a browser tab] is already loaded and has been interacted with [subject to browser permission policy]
    2. The user enters a suttaplayer sutta URL reference into the browser's address bar and initiates a HTTP request
  2. Via Goto URL
    1. The user clicks the Share button (3 connected-circles icon) on the top control bar to display the share drop-down panel
    2. Click on the Goto URL in system clipboard menu item
  3. Via the Catalog list, Search results list, or Playlist list
    1. The user clicks on a blue sutta link reference in a table row from a nominated list
  4. Via the audio player detecting a follow-on track & when Play next is toggled "on"
    1. The currently playing audio track reaches it's stop point
    2. The next queued track gets loaded and played
  5. Via the previous/nxt audio controls & when the Play next button is toggled "on"
    1. The user nominates to skip backward by clicking the previous track button, or skip forward by clicking the next track button. The previous and next buttons are found on the top control bar on either side of the Play/Pause button

6. How to search content for a sutta in the Search view?

On the first app load, approx 1350 sutta text documents (9MB) will get downloaded to the device to enable the app to fully function offline. This sutta content can be searched using the Search view.

There are two types of search: 1. Basic & 2. Advanced. The key difference is that Advanced searches use a regular expression as the search criteria entered in the Search criteria text box and must explicitly specify to ignore case & diacritics.

  1. Basic search uses the following default settings:

    1. Case insensitive allows one, for example, to enter Buddha or buddha in the search criteria and all subsequent text matching will be processed in lowercase
    2. Diacritics are ignored which allows one to enter Sāriputta or Sariputta and all subsequent text matching will be processed using only the standard character set:
      • DIACRITICS Characters = ["ā", "ī", "ū", "ṁ", "ṃ", "ṇ", "ṅ", "ñ", "ṣ", "ṭ", "ḍ", "ḷ", "ḥ"]
      • STANDARD Characters = ["a", "i", "u", "m", "m", "n", "n", "n", "s", "t", "d", "l", "h"]
    3. "And" specifiers honour the settings in the expanded search criteria drop-down panel

  2. Advanced search uses regular expression:

    1. Case insensitivity must be explicitly specified using the "i" regEx flag
    2. Diacritics will only get ignored if a custom "D" regEx flag is explicitly specified
    3. The "And" specifiers in the expanded search criteria drop-down panel are ignored
    4. Regular expressions are entered in the following format:
      • /<regEx-specifier>/[<regEx-flags>]

  3. Navigate to the Search view

  4. Click All to toggle "on" searching the entire catalog (ie. all albums)
  5. Select the start searching with Album beside the All toggle button. This selection is valid even when searching the entire catalog because it offers a quicker abort option when the desired reference(s) have already been found
  6. Click the expand criteria button (down arrow icon) beside the search criteria text box and take note of the "and" settings. The "and" drop-down panel is essential in order to find suttas where the text body includes references to term X and term Y and term Z
  7. Enter the search criteria in the search criteria text box
  8. Click the search button (search icon) to initiate the search.
    1. Search progress, pause & abort options will get displayed in the actions drop-down panel
    2. Search results will be displayed as rows in the table that match the search criteria

Eg. Basic multi-term search

Suppose one heard about a sutta where the Buddha and Venerable Sariputta discusss the causality of trade. To search for such a sutta consider using the following steps:

  1. Navigate to the Search view
  2. Click All to toggle "on" searching all albums
  3. Select Aṅguttara Nikāya as the start searching with Album option
  4. Click the expand criteria button (down arrow icon) beside the Search criteria text box and ensure:
    1. Apply "and" between terms is toggled "on"
    2. ";" is set as the "and" delimiter
  5. Enter sariputta;trade;failure in the Search criteria text box
  6. Click the blue Search button (search icon) beside the Search criteria text box
    • This should result in 19 results in 1 file(s)

Eg. Advanced regular expression search

Suppose one heard about a sutta where the Buddha encouraged the monks to learn from Venerable Sāriputta & Venerable Moggallāna. The find this sutta consider searching for where sariputta and moggallana occur within 10 words of one another using the following steps:

  1. Navigate to the Search view
  2. Click All to toggle "on" searching all albums
  3. Select Aṅguttara Nikāya as the starting searching with option
  4. Enter /(sariputta( [^ ]*){1,10} moggallana)/gmiD in the Search criteria text box
  5. Click the blue Search button (search icon) beside the Search criteria text box
    • This should result in 26 results in 7 file(s)

7. How to create a playlist of suttas in the Playlist view?

Playlists are arbitrary collections of sutta tracks, or perhaps just different sections from the same sutta track.

Creation

  1. Navigate to the Playlist view
  2. Click the playlists selection drop-down to list all existing playlists
  3. Select the first option titled "New playlist"
  4. Click on the text box beside the selection box to change the text to the preferred playlist name
  5. Click the Save button (disk icon) to save the newly named playlist

Population

The simplest way to populate playlists are to first source tracks of interest from either the Catalog, Search results or even other Playlists using the Copy selections menu item from the blue Actions button (3 vertical dots icon) on any of the aforementioned pages.

  • Catalog/Search results/Playlist
    1. Navigate to the view
    2. Click one-or-more checkboxes beside the track of interest to ensure that it now displays as checked
    3. Click on the blue Actions button (3 vertical dots) to display the actions drop-down panel
    4. Click the Copy selections (copy icon) menu item
    5. Navigate back to the newly created playlist outlined in the Creation section above
    6. Click on the blue Actions button (3 vertical dots) to display the actions drop-drown panel
    7. Click the Paste copied selections (paste icon) menu item and the track references will now be duplicated into the playlist and displayed

8. Some lists are too large and it's very difficult to find suttas

Indeed. There are over 400 sutta tracks in Saṁyutta Nikāya (aka SN) alone which can make finding difficult. Therefore, on every view page that has a table of tracks, finding is supported via 1. Find in page, & 2. Filtering track attributes:

  1. Find in page

    1. Click the Text button (T icon) from the top control bar to display the Text drop-down panel
    2. Enter the text in the Find in page edit box and click the search button
    3. Continue clicking the search button to jump to the next occurance of that text in the page until it gets to the last

  2. Filter the track attributes

    1. Click the blue Filter button (filter icon) to display a filter drop-down panel
    2. Enter the filter criteria by:
      1. Entering title text and clicking the Search button (search icon)
      2. Toggling zero or more track durations
      3. Toggling zero or more cache status (ie. was audio downloaded)
    3. The table rows are automatically updated to reflect only those matching the filter criteria

9. Why do sutta references indicate uncached audio (red dot) even after listening to the audio?

A green or red dot indicator is displayed beside every sutta reference across all views

  • Green indicates that the sutta has been cached via the offline feature
  • Red indicates that the sutta (even if listened to in full repeatedly) has not been explicitly downloaded via the offline feature

Audio and video content are treated differently to other content accessed over the internet due to size, segmentation and buffering issues. Such resources are streamed, and thus only partial content is registered as being received by browsers. Hence, the only way to ensure that the audio is available offline is to have the sutta audio downloaded via the offline feature.

10. How to download the sutta tracks of interest?

Applicable for Catalog/Search results/Playlist:

  1. Navigate to a list view
  2. Click one-or-more checkboxes beside the track of interest to ensure that it now displays as checked. Note, if one wants to (de)/select all, then click on the checkbox in the table header column
  3. Click on the Offline button (no-cloud icon) from the top control bar to display the offline drop-down panel
  4. Click the Download selections (down arrow icon) menu item and the nominated tracks will start downloading indicating progress with each download
  5. On completion those selection that were checked for download will appear with a green indicator beside them indicating that will be available even when the device is offline

11. How to create a bookmark with line highlighting, text marked up, start from line number and start/stop audio markers?

Many sutta's have a lot of redundancy built into them by design. Sharing a bookmark link of a specific passage with others can be cumbersome. Even for one's reference purposes it would beneficial if entire lines could be highlighted, phrases/words marked up, a specific line immediately scrolled to upon sutta load, or even just a specific segment of audio played. These are the key features that are all available when in the Reader view.

  1. Navigate to the Reader view with the nominated sutta displayed:

    • The sutta text body is displayed in the main section of the page with a gutter section to the left dedicated for line numbers. Each line from the sutta will display beside its associated line number. Note, lines will wrap dependent on the device's screen resolution; on mobile devices, expect lines to appear wrapped like paragraphs.

    • The sutta audio is displayed and managed via the Audio button (speaker icon) from the top control bar. Although, this drop-down panel is accessible under any page view, it is critical to specifying audio segment of interest. The first three rows all play a role for marking up an audio segment:

      1. 1st row - audio play head used to navigate time within the track
      2. 2nd row - skip forward/backward in nominated increments for accurate play head positioning
      3. 3nd row - Start & Stop buttons separated by '/' with time markers (in secs) to demark audio segments

  2. Line highlighting

    1. Scroll to the desired line location
    2. Click on the line number in the gutter section and the entire line will get highlighted

  3. Text markup

    1. Scroll to the desired segment of interest within the text body
    2. Click on the start of the word, and without releasing the mouse, drag the highlight cursor to the end of the word in the segment of interest and then release the mouse (or touch on mobile). In a desktop environment the segment of text will immediately be marked up; on mobile the segment will be marked up after a 3 second delay.

  4. Start from line number

    1. Scroll to the desired line location
    2. Double click on the line number in the gutter section and the line's row will be marked with a blue active indicator

  • Start/stop audio marker

    1. Click on the Audio button (speaker icon) in the top control bar to display the audio drop-down panel
    2. Using the audio control in row 1, drag the play head to close to the desired location
    3. Using the skip forward/back and the time entry box in row 2, fine-tune to the exact start position within the audio
    4. [Start] Click on the left time button in row 3 and the marked audio position from row 2 will now be registered as the marked audio start time. Press the same button again to
    5. [Stop] Click on the right time button in row 3 and the marked audio position from row 2 will now be registered as the marked audio stop time

  • Any accidental bookmark markings can be undone by using the browser's or app's back button/gesture. With each individual marking change the brower's or app's URL is updated

12. How to share a configured bookmark with others?

Having followed the instructions outlined in Question 11 above:

  1. [In Browser]
    1. Click in the browser's address bar and highlight the entire URL
    2. Use the operating system's Copy into clipboard feature (eg. Ctrl+C in Microsoft Windows)
  2. [In App]
    1. Click on the Share button (3 connected-circles icon) from the top control bar to display the share drop-down panel
    2. Click on Copy URL to clipboard

Note: The in App procedure will produce a URL with the parameter "&olo=1" appended to the end. "olo=1" signifies OnLine-Only as true and when used as an initial load in the browser, it will avoid downloading the 9MB of sutta text files.

14. How to get the audio and the text in the Reader view to sync while playing?

Note:, Maintaining a permanent sync between the text and the audio is not always desirable. On occasions there arises the desire to briefly revisit some text that was just heard by scrolling the page whilst letting the audio continue playing. Therefore, this feature can be toggled on and off in such instances.

  1. Navigate to the Reader view
  2. Click on the blue Actions button (3 vertical dots) to display the actions drop-down panel
  3. [Enable] Click the Auto scroll text with audio button to toggle "on"
    1. Every 3 seconds the approximate sutta text should display should get scrolled and centered into position
  4. [Disable] Click the Auto scroll text with audio button to toggle "off"

15. How to get the audio to approximately skip to the a position of interest within the text?

  1. Navigate to the Reader view
  2. Click the Pause button (pause icon) from the top control bar to avoid auto scroll interference
  3. Scroll the sutta text to the location of interest
  4. Click the mouse (or tap) in a letter in a position of interest and a information message will appear indicating that the cursor position was marked for audio sync. Note, on mobile, wait for 3 seconds before the information message to appear
  5. Click on the blue Actions (3 vertical dots) button to display the actions drop-down panel
  6. Click the Sync audio to cursor text position and the play head will move to an approximate position
  7. Click the Play button (play icon) from the top control bar to continue playing audio

16. How to share a playlist with others?

Sharing a playlists with others or between devices is achieved through transfer via 1. Clipboard, & 2. File system.

  1. Click on the Share button (3 connected circles icon)
  2. Click on the Transfer down-down list to select from either the clipboard or file
  3. Navigate to the Playlist view
  4. Click on the red Safety button (hard-hat icon) to display the safety drop-down panel
  5. Click the Export current playlist menu item
  6. Open email/message/social etc. and paste the contents from the clipboard
